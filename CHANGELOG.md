# 更新日志

## [1.0.1] - 2025-01-27

### 📁 文件管理优化
- 🎯 **默认保存目录更改**: 规则文件现在默认保存到 `.cursor/rules/` 目录下，而不是项目根目录
- 📂 **自动目录创建**: 如果 `.cursor` 或 `.cursor/rules` 目录不存在，插件会自动创建
- 🛠️ **目录结构管理**: 新增 `ensureCursorRulesDirectory()` 函数，统一管理目录创建逻辑
- 📋 **用户体验提升**: 避免在项目根目录产生过多文件，保持项目结构整洁
- 📖 **文档更新**: 更新README和项目概述文档，反映新的目录结构

### 技术改进
- 🔧 **代码重构**: 优化文件路径生成逻辑，提高代码可维护性
- 📝 **日志增强**: 添加目录创建过程的日志记录
- ✅ **功能测试**: 确保所有功能在新目录结构下正常工作

## [1.0.0] - 2025-06-07

### 🚀 正式版本发布
- 🎯 **功能完整**: 包含18个精心设计的开发规则模板，覆盖全栈开发技术栈
- 📱 **移动开发**: Android, iOS, Flutter, React Native, UniApp 完整支持
- 🌐 **Web前端**: React, Vue.js, Next.js, HTML/CSS, Chrome扩展
- 🖥️ **后端技术**: Node.js, Python, Go, FastAdmin/ThinkPHP
- 🎭 **AI助手角色**: 多种专业角色模板，提升开发效率

### 🏷️ 文件命名规范优化
- 📝 **统一命名规则**: 修改文件生成逻辑，现在所有规则文件都使用 `{template-name}-rules.mdc` 格式
- 📋 **命名示例**:
  - `android.mdc` 模板 → 生成 `android-rules.mdc` 文件
  - `react.mdc` 模板 → 生成 `react-rules.mdc` 文件
  - `vue.mdc` 模板 → 生成 `vue-rules.mdc` 文件
  - `nodejs.mdc` 模板 → 生成 `nodejs-rules.mdc` 文件
- 🎯 **一致性提升**: 确保所有18个模板都遵循相同的命名规范
- 👥 **用户体验**: 文件名更清晰地标识其为"规则文件"，避免与原始模板混淆

### 🎨 用户界面
- ✨ **分类菜单**: 按技术栈分组的清晰选择界面
- 🗂️ **智能分组**: 通用规则、前端、后端、移动开发、其他平台
- 🎯 **双重选择**: 支持用户角色规则和项目角色规则
- 📋 **模板预览**: 提供详细的模板描述和功能说明

### 技术特性
- 🛡️ **类型安全**: 完整的TypeScript类型定义
- 📊 **日志系统**: 完善的错误处理和调试功能
- 🔧 **代码质量**: 遵循最佳实践，通过ESLint检查
- ✅ **稳定性**: 经过充分测试，确保功能可靠
